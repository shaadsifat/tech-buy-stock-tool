Tech Buy BD

System Diagrams

Architecture, database, and behavior reference for the Tech Buy Stock Tool — a Flask + SQLite app that compares Tech Buy's prices and stock against competitor sites.

Dashboard — click Start Fetching to pull fresh prices/stock from every listed product's Other-site link (Tech Buy's own price/stock comes from the Shopify catalog sync, not this fetch). While a fetch is running, Stop Fetching appears — clicking it halts things almost immediately. The stat cards, charts, and Quick Summary box all update live while fetching, with no need to reload the page. Once done, use Import in Excel File to export the full report.

No Price Sync — some products (usually official-distributor items) have a price set manually and don't need competitor comparison at all. Flagging them pulls them out of the whole comparison workflow entirely: hidden from the product lists, excluded from the Dashboard's counts/charts, and skipped by Start Fetching. Their Other-site link, if any, is kept on record, just ignored.

Single Product List and Variant Products — every product is tracked, searched, filtered, and sorted, with products that have color/size/etc. options broken down variant by variant rather than as one lump product, so a mismatch on just one option doesn't get missed inside an otherwise-fine listing.

How flagging works — Need Action = Yes means either the price differs by more than the allowed tolerance, or Tech Buy's stock status doesn't match the Other site's. Reviewed gets ticked once the issue is manually fixed on Tech Buy's store, and auto-locks itself when Need Action is already No — nothing to review.

Sync Tracking — a dedicated page listing everything that needs follow-up from the Shopify sync: products missing an Other-site link, products removed from Shopify, and products with an incomplete set of variant options.

Shopify catalog sync — Tech Buy's own product data (name, price, stock, variants) comes from Shopify's own API instead of scraping the storefront page. This runs as a separate script, not a button in the software — run it manually whenever you want to refresh the catalog. It updates every existing product's Tech Buy name/price/stock without touching the Other-site link or Reviewed status already set up, and adds new products with the Other-site link left blank, ready to pair up.

02

Database Schema

Four tables. A variant-parent product gets one products row plus one product_variants row per variant; a single-variant product gets one products row plus one fetch_results row instead. A product_variants row with shopify_variant_id = NULL is a synthetic "Found on Other site, missing from Tech Buy" row, not a real Shopify variant.

  actor User
  participant UI as Dashboard (JS)
  participant Flask as Flask Route

  UI->>Flask: POST /fetch/start
  Flask->>Runner: start_fetch()
  activate Runner
  Runner->>Models: get_all_products()
  Models->>DB: SELECT products
  DB-->>Models: rows
  Models-->>Runner: product list

  loop each product, thread pool
    Runner->>Parser: fetch(other_link)
    Parser-->>Runner: price / stock
    Runner->>Models: update_*_fetch_result()
    Models->>DB: UPSERT fetch_results / product_variants
  end
  deactivate Runner

  loop every 1.5s while running
    UI->>Flask: GET /dashboard/stats
    Flask->>Models: get_dashboard_data()
    Models->>DB: aggregate queries
    Models-->>Flask: stats, report, summary
    Flask-->>UI: JSON
    UI-->>User: live-updating tiles and rows
  end
