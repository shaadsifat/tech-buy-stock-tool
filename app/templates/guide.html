Guide

How to use the software, and a short technical overview.

User Guide

Sidebar — click the small arrow on its edge to collapse it into a thin icon-only ribbon (the logo switches to the favicon to fit), or click again to expand it. Your choice is remembered across visits.

Dashboard — click Start Fetching to pull fresh prices/stock from every listed product's Other-site link (Tech Buy's own price/stock comes from the Shopify catalog sync, not this fetch — see below). While a fetch is running, Stop Fetching appears — clicking it halts things almost immediately (anything already mid-request finishes, anything not yet started is skipped). The 5 stat cards, the charts, and the Quick Summary box all update live while fetching, with no need to reload the page. Once done, use Import in Excel File to export the full report, or Remove Previous Fetched Data to clear the Other-site results before a fresh run (Tech Buy's own price/stock is never affected by this, since it doesn't come from this fetch). The Quick Summary box is click-to-copy plain text, ready to paste to management.

The stat cards: Total Products, Categories, Need Action, Fetch Problems (Other-site fetches that failed), and Perfectly Fetched (Other-site fetches that succeeded cleanly).

Other Link Upload — products themselves only ever come from the Shopify catalog sync now; there's no manual one-by-one "add product" anymore. This page's Bulk Upload only attaches/updates the Other-site link on a product that already exists, matched by its Shopify Handle (never by name, which isn't guaranteed unique or stable) — the handle is pulled straight out of the Tech Buy Link column (case-insensitive), since every Tech Buy Link ends in /products/<handle>. Upload an Excel file with columns Category, Product Name, Tech Buy Link, Other website link, in that exact order (Export Listed Products downloads your current list in this exact format, ready to edit and re-upload) — Category/Product Name are only ever used for display in the failure report, never for matching. Each row is checked and reported individually:

  • Link not matched — the Tech Buy Link doesn't look like a real product link, so no handle could be read from it.
  • New product — a handle was read fine, but it isn't in the database yet. Bulk upload can't create it; it has to come from the Shopify sync first.
  • Duplicate — that handle already appeared earlier in the same file.

No Price Sync — some products (usually official-distributor items) have a price you set yourself and don't need competitor comparison at all. Tick them here (individually, or via the bulk bar's Mark/Unmark) and they're pulled out of the whole comparison workflow entirely: hidden from Single Product List, Variant Products, and Sync Tracking's Missing Other Link tab, excluded from the Dashboard's counts/charts, and skipped by Start Fetching. Their Other-site link (if any) is kept on record, just ignored — untick to bring a product back into normal tracking. The page header shows how many products are currently flagged, and the No Price Sync column has its own filter (funnel icon) like every other filterable column in the app.

Single Product List (products with 0-1 real Shopify variants, and not flagged No Price Sync) — search by name, filter any column via the funnel icon on its header (or the Category / Other Site dropdowns), sort by clicking a header, and pick which columns show via Customize Columns (Category and Shopify Status are hidden by default, still available there). Click any cell to copy its text. Every filter dropdown only ever shows values that actually occur in your data right now — if nothing is currently a Draft, "Draft" simply won't appear as a choice. Column filters are remembered across visits — they'll still be applied next time you open this page, even after navigating away. Rows update live as a fetch progresses (no reload needed), whether it was started from here or from the Dashboard, and a Stop Fetching control appears here too whenever one is running.

If a paired Other-site link quietly grows variants after you originally paired it (common when a single-listing competitor product later gets a color/size option added), fetching detects this: Stock Status (Other) shows Variant Found, Need Action = Yes, and Reviewed becomes editable, so you know to move that product over to proper variant tracking. If the Other site still only has exactly one real variant, nothing changes.

Bulk actions (Single Product List and Variant Products both have this bar) — tick the checkbox on the left of any row, or use the checkbox in the header to select every row currently shown on the page. A bar appears with:

  • Mark as Reviewed — ticks Reviewed for the selected products (skips any already auto-locked because Need Action is No). On Variant Products this ticks every variant under each selected product instead, since a product only counts as Reviewed once all of its variants are.
  • Re-Fetch Selected — runs a fresh Other-site price/stock fetch for only the selected products, instead of the full list.
  • Export Selected — downloads an Excel file containing only the selected products, in the same format as the full export.
  • Delete Permanently — removes the selected products (and, on Variant Products, all their variants) for good. Asks for confirmation first, since this can't be undone.

Variant Products (products with 2+ real Shopify variants, and not flagged No Price Sync) — one row per product, click the arrow to expand its full variant breakdown in a nested table underneath. The outer row shows Name, Tech Buy Link, Other Link (there's only ever one Other-site link per product, even with many variants), Shopify Status, an aggregate Fetched Status/Need Action/Reviewed — all computed from that product's variants (Need Action/Fetched Status = "bad" state if any variant has it, "clean" state only once every variant does; Reviewed = true only once every variant is individually reviewed). Ticking a variant's Reviewed checkbox updates the product's aggregate Reviewed status immediately, no reload needed. Customize Variant Columns controls the inner table only (defaults to Variant, Stock (Other), both Regular/Sale prices, Need Action, and Reviewed — everything else, including Match, is available there too).

Variant matching — each Tech Buy variant is matched against the Other site purely by variant name, as an order-independent set: for a 2-layer option like Color × Material, "Red > Silver" and "Silver > Red" (or any other ordering, with any number of shared layers) count as the exact same variant. The inner table's Match column shows one of 4 states per variant:

  • Found — exists by name on both sites. Price/stock differences still drive Need Action/Reviewed exactly like the main comparison logic.
  • Not Found — exists on Tech Buy (In Stock) but not on the Other site. Need Action = Yes, Reviewed is editable.
  • Found TB — exists on Tech Buy, but it's Out of Stock there and missing on the Other site too — normal, not actionable. Need Action = No, Reviewed locks automatically.
  • Found OT — exists on the Other site but not on Tech Buy at all. Gets its own row in the inner table (same columns, fully interactive), Need Action = Yes, Reviewed editable — usually means Tech Buy is missing a variant the competitor already carries.

A product only counts as fully Reviewed once every one of its variants is — including any Found OT rows.

Above the table, a tab bar sorts every variant-parent product into exactly one bucket, checked in this order (first match wins):

  • Exact Match — every variant is Found or Found TB. Nothing to chase down.
  • Variant Not Found — at least one variant is Not Found (checked only among products that didn't already qualify for Exact Match).
  • Missing from Tech Buy — at least one variant is Found OT (checked only among products left after the first two buckets).

Categories and Other Websites — manage categories, and see how many products point to each "Other" site.

Sync Tracking — a dedicated page listing everything that needs follow-up from the Shopify sync, across 3 tabs: products missing an Other-site link, products removed from Shopify (only ones that had an Other-site link to begin with — untracked products dropping off Shopify isn't a follow-up item), and products with a variant issue. Every cell is click-to-copy, Stock (Tech Buy) can be sorted and filtered, and Category has its own dropdown. Click the edit icon on any row to jump straight to fixing it.

Shopify catalog sync — Tech Buy's own product data (name, price, stock, variants) comes from Shopify's own API instead of scraping the storefront page, since the page could sometimes serve a stale cached price to automated requests. This runs as a separate script (get_product_api_call.py), not a button in this software — run it manually whenever you want to refresh the catalog (e.g. every few days, or after adding new products in Shopify). It:

  • Before pulling anything, purges every local product that has no Other-site link and isn't flagged No Price Sync — these were never paired with anything, so they're cleared out and re-created fresh from Shopify if still Active. Only paired products (real Other-site link) and intentionally-unpaired ones (No Price Sync) survive a re-sync untouched.
  • Only syncs Active products (in stock or out of stock, doesn't matter — just not Draft). A Draft product is skipped entirely, and if it was already in the database from an earlier sync, it gets flagged Removed instead of silently sticking around.
  • Updates every existing product's Tech Buy name/price/stock, without ever touching the Other-site link or Reviewed status you've already set up.
  • Adds new products it finds, with the Other-site link left blank — pair it up via Other Link Upload or by editing the product directly.
  • Shopify Status column — shows Active for anything currently live on Shopify, or Removed if a product was previously synced but is no longer found Active (unpublished, deleted, or turned Draft) — so nothing just silently disappears.
  • Variant Issue column — flags a product as Incomplete if its variants don't cover every combination of its own options (e.g. a 3×3 Switch/Color keyboard that only has 8 of the 9 combinations) — hover the badge to see exactly which combination is missing. Usually means someone forgot to finish setting up that product in Shopify.
  • After running the sync, click Start Fetching as normal to pull the Other-site prices and compute Need Action — Tech Buy's side is never fetched by this button at all anymore, only the Other site.

Fetched Status — since Tech Buy's price/stock no longer comes from fetching, this column now reflects the Other-site fetch only, and only has 3 possible values:

  • Fetched — the Other-site page was reached successfully.
  • Fetch Problem — the Other-site page couldn't be reached or parsed.
  • (blank/—) "Not Fetched" — no Other-site link is paired yet, so there was nothing to fetch (see Sync Tracking's "Missing Other Link" tab).

How flagging works:

  • Need Action = Yes means either the price differs by more than the allowed tolerance, or Tech Buy's stock status doesn't match the Other site's — either way, something needs a manual fix on Tech Buy's store.
  • Reviewed (checkbox) — tick it once you've manually fixed the issue. It auto-ticks and locks itself when Need Action is already No — nothing to review. If Need Action is Yes (price or stock mismatch), it stays editable so you can tick it once fixed.
  • Clicking Start Fetching resets every Reviewed checkbox first, since a fresh fetch makes the old review status outdated.
  • In the exported Excel file, the Updated column shows "Yes" if a product is Reviewed, even if the last fetch still shows Need Action = Yes — it reflects your manual fix until the next fetch confirms it.

Allowed price difference (tolerance) before flagging: the bracket is picked using Tech Buy's own price (its Sale Price if it has one, else its Regular Price).

Tech Buy Price Allowed Difference
$150 or moreUp to $3
$100 to $149.99Up to $1
Under $100$0 — must match exactly

If a product has both a Regular and a Sale price: the Sale Price is checked against the table above. The Regular Price is checked separately with a flat $100 allowance regardless of price. If there's no Sale Price, only the Regular Price is checked against the table above.

On top of the price check, Stock Status (Tech Buy) and Stock Status (Other) must also be identical (e.g. both "In Stock"). Need Action = Yes if the price check fails, the stock statuses don't match, or both.

Exception: if both sides show "Out of Stock", price is skipped entirely — Need Action = No automatically, since nobody has it in stock to compare.

Technical Guide

  • Backend: Python (Flask), served by the Waitress production WSGI server.
  • Database: SQLite, a single local file at data/app.db — no external database server needed.
  • Frontend: server-rendered HTML/CSS with vanilla JavaScript — no framework.
  • Other-site price fetching: requests + BeautifulSoup, one parser per website, run concurrently across products. Each site gets a polite gap between requests of 1 second, randomly varying up to ±15% each time (so ~0.85s–1.15s, never a perfectly regular bot-like cadence) — a single manual re-fetch of one product skips this entirely, since it's not a burst.
  • Tech Buy catalog data: pulled directly from Shopify's Admin GraphQL API (read-only access) via the separate get_product_api_call.py script, instead of scraping — sidesteps a caching/bot-detection issue on the storefront that could otherwise serve stale prices. Credentials live in a local, git-ignored .env file, never committed.
  • Excel import/export: openpyxl.
  • Runs entirely on the local PC — no data leaves the machine unless you export or push it to GitHub yourself. Version-controlled with Git.
