Guide

How to use the software, and a short technical overview.

User Guide

Dashboard — click Start Fetching to pull fresh prices/stock for every listed product. Once done, use Import in Excel File to export the full report, or Remove Previous Fetched Data to clear results before a fresh run. The Quick Summary box is click-to-copy plain text, ready to paste to management. Below that are stat charts and, further down, live counts by status.

Input Product — add one product at a time, or use Bulk Upload to import many from an Excel file (Category, Product Name, Tech Buy Link, Other website link — unknown categories are created automatically). Export Listed Products downloads your current list in that same format.

Product List — search by name, filter any column via the funnel icon on its header (or the Category / Other Site dropdowns), sort by clicking a header, and pick which columns show via Customize Columns. Click any cell to copy its text.

Categories and Other Websites — manage categories, and see how many products point to each "Other" site.

How flagging works:

  • Need Action = Yes means Tech Buy's price and the Other site's price differ by more than the allowed tolerance — it needs a manual price correction on Tech Buy's store.
  • Reviewed (checkbox) — tick it once you've manually fixed the price. If Need Action is already No (prices already match), the checkbox auto-ticks and locks itself — there's nothing to review.
  • Clicking Start Fetching resets every Reviewed checkbox first, since a fresh fetch makes the old review status outdated.
  • In the exported Excel file, the Updated column shows "Yes" if a product is Reviewed, even if the last fetch still shows Need Action = Yes — it reflects your manual fix until the next fetch confirms it.

Allowed price difference (tolerance) before flagging: the bracket is picked using Tech Buy's own price (its Sale Price if it has one, else its Regular Price).

Tech Buy Price Allowed Difference
$150 or moreUp to $3
$100 to $149.99Up to $1
Under $100$0 — must match exactly

If a product has both a Regular and a Sale price: the Sale Price is checked against the table above. The Regular Price is checked separately with a flat $100 allowance regardless of price. Need Action = Yes if either check fails. If there's no Sale Price, only the Regular Price is checked against the table above.

Technical Guide

  • Backend: Python (Flask), served by the Waitress production WSGI server.
  • Database: SQLite, a single local file at data/app.db — no external database server needed.
  • Frontend: server-rendered HTML/CSS with vanilla JavaScript — no framework.
  • Price fetching: requests + BeautifulSoup, one parser per website, run concurrently across products.
  • Excel import/export: openpyxl.
  • Runs entirely on the local PC — no data leaves the machine unless you export or push it to GitHub yourself. Version-controlled with Git.
