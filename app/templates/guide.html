Guide

How to use the software, and a short technical overview.

User Guide

Sidebar — click the small arrow on its edge to collapse it into a thin icon-only ribbon (the logo switches to the favicon to fit), or click again to expand it. Your choice is remembered across visits.

Dashboard — click Start Fetching to pull fresh prices/stock for every listed product. While a fetch is running, Stop Fetching appears — clicking it halts things almost immediately (anything already mid-request finishes, anything not yet started is skipped). The 5 stat cards, the charts, and the Quick Summary box all update live while fetching, with no need to reload the page. Once done, use Import in Excel File to export the full report, or Remove Previous Fetched Data to clear results before a fresh run. The Quick Summary box is click-to-copy plain text, ready to paste to management.

The stat cards: Total Products, Categories, Need Action, Fetch Problems (TechBuy Prob + Other Prob + Both Prob combined — products where a fetch failed on one or both sides), and Perfectly Fetched (products fetched cleanly with no problems at all).

Input Product — add one product at a time, or use Bulk Upload to import many from an Excel file (Category, Product Name, Tech Buy Link, Other website link — unknown categories are created automatically). Export Listed Products downloads your current list in that same format.

Product List — search by name, filter any column via the funnel icon on its header (or the Category / Other Site dropdowns), sort by clicking a header, and pick which columns show via Customize Columns. Click any cell to copy its text. Column filters are remembered across visits — they'll still be applied next time you open this page, even after navigating away. Rows update live as a fetch progresses (no reload needed), whether it was started from here or from the Dashboard, and a Stop Fetching control appears here too whenever one is running.

Bulk actions — tick the checkbox on the left of any row, or use the checkbox in the header to select every row currently shown on the page. A bar appears with:

  • Mark as Reviewed — ticks Reviewed for the selected products (skips any already auto-locked because Need Action is No).
  • Re-Fetch Selected — runs a fresh price/stock fetch for only the selected products, instead of the full list.
  • Export Selected — downloads an Excel file containing only the selected products, in the same format as the full export.
  • Delete Permanently — removes the selected products (and their fetched data) for good. Asks for confirmation first, since this can't be undone.

Categories and Other Websites — manage categories, and see how many products point to each "Other" site.

How flagging works:

  • Need Action = Yes means either the price differs by more than the allowed tolerance, or Tech Buy's stock status doesn't match the Other site's — either way, something needs a manual fix on Tech Buy's store.
  • Reviewed (checkbox) — tick it once you've manually fixed the issue. It auto-ticks and locks itself when Need Action is already No — nothing to review. If Need Action is Yes (price or stock mismatch), it stays editable so you can tick it once fixed.
  • Clicking Start Fetching resets every Reviewed checkbox first, since a fresh fetch makes the old review status outdated.
  • In the exported Excel file, the Updated column shows "Yes" if a product is Reviewed, even if the last fetch still shows Need Action = Yes — it reflects your manual fix until the next fetch confirms it.

Allowed price difference (tolerance) before flagging: the bracket is picked using Tech Buy's own price (its Sale Price if it has one, else its Regular Price).

Tech Buy Price Allowed Difference
$150 or moreUp to $3
$100 to $149.99Up to $1
Under $100$0 — must match exactly

If a product has both a Regular and a Sale price: the Sale Price is checked against the table above. The Regular Price is checked separately with a flat $100 allowance regardless of price. If there's no Sale Price, only the Regular Price is checked against the table above.

On top of the price check, Stock Status (Tech Buy) and Stock Status (Other) must also be identical (e.g. both "In Stock"). Need Action = Yes if the price check fails, the stock statuses don't match, or both.

Exception: if both sides show "Out of Stock", price is skipped entirely — Need Action = No automatically, since nobody has it in stock to compare.

Technical Guide

  • Backend: Python (Flask), served by the Waitress production WSGI server.
  • Database: SQLite, a single local file at data/app.db — no external database server needed.
  • Frontend: server-rendered HTML/CSS with vanilla JavaScript — no framework.
  • Price fetching: requests + BeautifulSoup, one parser per website, run concurrently across products. Each site gets a polite, randomized gap between requests (a single manual re-fetch of one product skips this, since it's not a burst) to avoid being flagged as bot traffic.
  • Excel import/export: openpyxl.
  • Runs entirely on the local PC — no data leaves the machine unless you export or push it to GitHub yourself. Version-controlled with Git.
